### 13. No JOIN of `book_events` against a `listening_sessions` aggregate — PASS

The only `JOIN book_events` in the codebase is [db.py:467](../src/fabulor/db.py#L467), inside `get_finished_book_data`:

```sql
SELECT b.id, MAX(be.event_time) as last_finished
FROM books b
JOIN book_events be ON be.book_id = b.id
WHERE b.is_deleted = 0 AND b.is_excluded = 0 AND b.is_missing = 0
  AND be.event_type = 'finished'
GROUP BY b.id
```

This aggregates `book_events` against `books` — **`listening_sessions` does not appear in the query**, so there is no sessions×events cartesian product and no `SUM(listened_seconds)` inflation. The forbidden shape (joining `book_events` into a query that aggregates `listening_sessions`) is absent; the queries that *do* need a finished flag alongside session sums use the sanctioned correlated subquery instead. No violation.
