## 1. `finished` written exactly once per EOF — (a) correct

The write is gated by `_eof_event_written` ([app.py:1435](src/fabulor/app.py#L1435)):
```python
if not self._eof_event_written and self._current_book is not None:
    self.db.write_book_event(self._current_book.path, 'finished', book_id=self._current_book.id)
    self._eof_event_written = True
```
The flag is set True immediately after the write ([app.py:1437](src/fabulor/app.py#L1437)) and is reset to False only on book load (`load_book` path, [app.py:1163](src/fabulor/app.py#L1163)) and at init ([app.py:300](src/fabulor/app.py#L300)). A user pausing at EOF, resuming, and re-hitting EOF on the **same** book cannot re-enter the write block — the flag is still True (it only clears on a new `load_book`). One finished event per EOF-of-a-loaded-book. ✅

## 2. EOF `book_events` write populates `book_id` — (a) correct

There is exactly **one** `write_book_event` call site in the entire app outside db.py: [app.py:1436](src/fabulor/app.py#L1436), which passes `book_id=self._current_book.id` (non-NULL — guarded by the `self._current_book is not None` condition at [app.py:1435](src/fabulor/app.py#L1435)). Grep for `write_book_event(` across `src/fabulor/` returns only the definition ([db.py:850](src/fabulor/db.py#L850)) and this single caller. The previously-problematic third write path that produced `book_id=NULL` rows has **no sibling or regression** — no other emitter exists. ✅

## 3. `unfinish_book()` deletes only the most recent finished event — (a) correct

[db.py:857-867](src/fabulor/db.py#L857-L867):
```sql
DELETE FROM book_events
WHERE id = (
    SELECT id FROM book_events
    WHERE book_id = ? AND event_type = 'finished'
    ORDER BY event_time DESC LIMIT 1
)
```
The subquery resolves to a single `id` (the most recent finished event for that `book_id`) via `ORDER BY event_time DESC LIMIT 1`, and the outer `DELETE WHERE id = (...)` removes exactly that one row. Under finished→reverted→finished-again: each EOF inserts a new finished row; each revert deletes only the newest, leaving prior finished events intact. No mass delete by `book_id`. ✅

## 4. `_eof_book_id` lifecycle — (a)* correct (app-close nuance)

**Set:** only at banner-show time, [app.py:1438](src/fabulor/app.py#L1438) `self._eof_book_id = self._current_book.id if self._current_book else None`, inside the same `not self._eof_event_written` block that shows the banner ([app.py:1439-1446](src/fabulor/app.py#L1439-L1446)). It is set nowhere else.

**Cleared** on every documented dismiss trigger:
- **New book load** — `load_book` reset [app.py:1122](src/fabulor/app.py#L1122) `self._eof_book_id = None`, plus the explicit `_dismiss_eof_prompt()` at [app.py:1104](src/fabulor/app.py#L1104).
- **Seek away from EOF** — `_dismiss_eof_prompt()` at [app.py:1467](src/fabulor/app.py#L1467) ("Left EOF (user seeked/rewound away)").
- **Restart** — `_dismiss_eof_prompt()` at [app.py:2090](src/fabulor/app.py#L2090) (the `eof_reached or text == "Restart"` branch).
- **Sleep timer start** — `_dismiss_eof_prompt()` at [app.py:658](src/fabulor/app.py#L658).
- **Scan start / folder add-remove-rescan** — inlined clear at [app.py:781](src/fabulor/app.py#L781) (`self._eof_book_id = None` when `show_cancel is True`, documented as the same contract as `_dismiss_eof_prompt`).
- **Explicit close button** — `_dismiss_eof_prompt` wired at [app.py:371](src/fabulor/app.py#L371).
- **Revert** — `_on_revert_finish` clears at [app.py:817](src/fabulor/app.py#L817).

`_dismiss_eof_prompt` / `_on_revert_finish` both null `_eof_book_id` before any revert can be retriggered ([app.py:837](src/fabulor/app.py#L837), [app.py:817](src/fabulor/app.py#L817)), and `_on_revert_finish` guards on `if self._eof_book_id is not None` ([app.py:815](src/fabulor/app.py#L815)). A stale id cannot apply a revert to the wrong book: every book transition either resets it (`load_book`) or dismisses it first.

**Nuance (see follow-up):** `closeEvent` ([app.py:2412](src/fabulor/app.py#L2412)) does **not** call `_dismiss_eof_prompt`. This is benign — the finished event is already persisted, the prompt is transient UI, and the process is exiting — so there is no stale-revert risk. The audit item lists "app close" as a clear trigger; functionally the state is discarded by process exit rather than explicitly cleared. Flagged low. ✅*

## 8. `_finished_dates` sync after revert — (a) correct

`_finished_dates` is replaced wholesale (not patched) on every refresh: `self._book_model.set_finished_dates(self.db.get_finished_book_data())` ([library.py:463](src/fabulor/ui/library.py#L463)), and `set_finished_dates` assigns `self._finished_dates = dates` ([library.py:842-843](src/fabulor/ui/library.py#L842-L843)). `get_finished_book_data` ([db.py:405-424](src/fabulor/db.py#L405-L424)) only returns books with a **surviving** finished event (`JOIN book_events ... WHERE event_type='finished'`). After `unfinish_book` deletes the row, the reverted book no longer appears in the query result, so the new dict has **no entry** for it — not a stale timestamp. The revert flow triggers `self.library_panel.refresh()` ([app.py:828](src/fabulor/app.py#L828)) which rebuilds the dict, and `stats_panel.refresh_all()` ([app.py:827](src/fabulor/app.py#L827)) which re-queries finished data for the FinishedScrollRow. No stale date can leak. ✅

## 9. Sort-field filter before text filter, incl. "finished" — (a) correct

`_apply_filter_and_sort` ([library.py:900](src/fabulor/ui/library.py#L900)) narrows `source` by sort field **first** ([library.py:905-910](src/fabulor/ui/library.py#L905-L910)):
```python
if self._sort_field == "last_played":
    source = [b for b in self._books if (b.progress or 0.0) > MIN_PROGRESS]
```

**then** applies text/tag/year filtering on `source` ([library.py:912-954](src/fabulor/ui/library.py#L912-L954)). The "finished" computed key is explicitly handled in the narrowing branch (line 908) — not just numeric/date keys. Order holds. ✅

## 10. Null-sort helper sends never-finished last in both directions — (a) correct

`effective_val` ([library.py:959-972](src/fabulor/ui/library.py#L959-L972)) returns `self._finished_dates.get(b.id)` (→ `None` for never-finished) for the "finished" field ([library.py:960-961](src/fabulor/ui/library.py#L960-L961)). The partition ([library.py:986-990](src/fabulor/ui/library.py#L986-L990)):
```python
have    = [b for b in books if effective_val(b) is not None]
missing = [b for b in books if effective_val(b) is None]
have.sort(key=sort_key, reverse=reverse)
missing.sort(key=lambda b: (b.title or "").lower())
books = have + missing
```
`missing` is always appended **after** `have` and sorted by title — independent of `reverse`. So never-finished books land last in both ascending and descending order. The same `effective_val`/partition applies to progress and last_played and the generic fields, so the secondary (title) sort on the missing partition is uniform. ✅

## 12. Three archived refresh paths re-render grayscale on flag toggle — (a)‡ correct (un-archive nuance)

- **Book detail:** `_refresh_archived_state` ([book_detail_panel.py:1204](src/fabulor/ui/book_detail_panel.py#L1204)) recomputes `_is_archived` from a fresh `get_book_dict` ([book_detail_panel.py:1206](src/fabulor/ui/book_detail_panel.py#L1206)) and re-applies the cover ([book_detail_panel.py:1220-1222](src/fabulor/ui/book_detail_panel.py#L1220-L1222)). Invoked from `_on_confirm_remove` ([book_detail_panel.py:1192](src/fabulor/ui/book_detail_panel.py#L1192)) when a book is excluded from a non-library context.
- **Stats (BookDayRow / FinishedBookThumb / FinishedScrollRow):** the exclude flows call `stats_panel.refresh_all()` ([app.py:827](src/fabulor/app.py#L827), [app.py:1449](src/fabulor/app.py#L1449)) or `refresh_current_tab()` ([app.py:862](src/fabulor/app.py#L862)), which re-feeds rows whose dicts now carry updated `is_deleted`/`is_excluded` → fresh `_is_archived` at construction (item 11). The `FinishedScrollRow` rebuild fires because its signature includes both flags (item 13).
- **Tag manager:** `tags_panel.refresh_books()` ([app.py:861](src/fabulor/app.py#L861)) reconstructs `_TagBookThumb`s with refreshed dicts.

A finished book later archived: `refresh_all` rebuilds the FinishedScrollRow (its `is_deleted`/`is_excluded` change the signature → rebuild → new thumbs grayscale). ✅

**‡ Nuance (see follow-up):** `_refresh_archived_state` re-applies the **currently displayed** pixmap ([book_detail_panel.py:1220-1222](src/fabulor/ui/book_detail_panel.py#L1220-L1222)). Re-grayscaling an already-gray pixmap is idempotent, so **archiving** renders correctly. But **un-archiving** (e.g. location re-add resurrecting a book while the detail panel is open) cannot restore color from an already-grayscaled pixmap — `to_grayscale` is one-way and the source color is gone. The stats/tag paths are immune (they reconstruct from the original cover file). Flagged medium because un-archiving a book with the detail panel live is a narrow but reachable path.

**Row staleness guard:** the prompt names `FinishedScrollRow._current_ids`; the actual member is **`_current_sig`** ([stats_panel.py:686](src/fabulor/ui/stats_panel.py#L686), [stats_panel.py:724-726](src/fabulor/ui/stats_panel.py#L724-L726)). This is **stronger** than an id-only guard: a finished/unfinished toggle changes membership (book_id set) and an archive toggle changes `is_deleted`/`is_excluded` — both force a rebuild, so a stale row from a previous filter state cannot linger. `_current_ids` as literally named is **(c) not found**, but the superior `_current_sig` satisfies the invariant's intent. ✅§

## Findings requiring follow-up

These are **notes**, not fixes-in-place (audit-only task). None is a hard `(b)` violation.

1. **[low] `closeEvent` does not clear `_eof_book_id` (item 4).** Benign today — the finished event is persisted and the process exits, so there is no stale-revert risk. If `closeEvent` ever gains a path that persists or restores transient banner state across restarts, add an explicit `_dismiss_eof_prompt()` (or `_eof_book_id = None`) to `closeEvent` ([app.py:2412](src/fabulor/app.py#L2412)) for symmetry with the other dismiss triggers.

2. **[medium] Un-archive can't restore color in the open detail panel (item 12).** `_refresh_archived_state` re-applies the already-displayed (possibly grayscaled) pixmap ([book_detail_panel.py:1220-1222](src/fabulor/ui/book_detail_panel.py#L1220-L1222)). Archiving is correct (idempotent grayscale); un-archiving a book while its detail panel is open leaves the cover gray because `to_grayscale` is lossy. Fix path when prioritized: reload the cover from `get_active_cover_path(self._book_path)` (as the load path at [book_detail_panel.py:587](src/fabulor/ui/book_detail_panel.py#L587) does) inside `_refresh_archived_state` instead of re-wrapping `self._cover_label.pixmap()`. Narrow trigger (location re-add with panel live), hence medium not high.

3. **[informational] Naming drift vs. audit text.** (a) The revert-button QSS lives in `get_base_stylesheet`, not `get_player_stylesheet` as the item phrased it — correct per the banner's `main_window` ownership (item 5). (b) The FinishedScrollRow staleness guard is `_current_sig`, not `_current_ids` (item 13) — a richer and stronger signature. Both are implementation-correct; only the audit checklist wording was stale. No code change needed; noted so a future audit does not re-flag them.
